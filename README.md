# 腫瘤指引快速查 App

## NCCN 分類與首頁狀態

- App 保留 93 個便於臨床搜尋的癌別卡片；NCCN 的合併指引可同時覆蓋多個細分類。
- 目前有 91 個癌別屬於 NCCN 對應範圍；「罕見癌症」與「其他未分類腫瘤」是自訂分類，不列入缺少 NCCN PDF 的計數。
- 首頁的「已有指引／缺 PDF」只依目前版本 PDF 判斷；編輯頁的「內容整理狀態」是人工 checklist、章節與精準欄位的整理進度，兩者互不混用。
- 已匯入的 NCCN PDF 會在 App 啟動時依檔名、標題與原本癌別關聯，自動補上同一合併指引涵蓋的細分類，不必重新匯入。
- 單筆或批次匯入可辨識的 NCCN PDF 時，畫面會列出自動對應的所有癌別；一份 PDF 覆蓋多張癌別卡片是正常情況。


個人本機使用之腫瘤指引查閱、少見癌症防漏、快速參考與本機指引文件庫。

## 健保 PDF 自動解析

- 匯入健保第 9 節或抗癌瘤藥物給付規定後，可在 PDF 文件庫點「解析健保規定」。
- App 使用本機 PDF.js 擷取文字，不會把 PDF 上傳；解析時會顯示頁面進度。
- 系統依 9.x 藥物條目與癌別名稱建立「PDF 自動擷取」健保資料，保留來源文件、條號及頁碼。
- 解析時會記錄公告／生效日期、來源文件、版本、條號與頁碼；新版同藥物規定會成為目前版本，舊的自動擷取資料改為封存而不刪除，手動資料不受影響。
- 表格、掃描頁、跨頁或無法對應癌別的條目會列入解析摘要，必要時仍需人工核對。自動擷取不等於個案已通過給付資格審查。

## 功能特色

- 🔍 **全癌別搜尋**：支援中文、英文、同義詞模糊搜尋
- 📄 **本機 PDF 文件庫**：匯入、管理、版本控制指引 PDF
- 🌐 **來源中心**：集中健保第 9 節抗癌瘤藥物、整份給付規定與 NCCN 官方入口
- ✅ **防漏 Checklist**：自動預載共同與癌別專屬必查項目，可編輯、可分類
- 🎯 **精準化欄位**：依癌別預載分期、病理、PD-L1 與相關基因欄位，留白不阻擋輸出
- 🏥 **選藥後查健保**：快速參考先選藥物／療程，再展開限制；缺少必要條件時可就地補填並重新比對
- 📋 **快速參考**：一鍵產生參考方向，顯示下一步建議
- 💾 **本機儲存**：所有資料儲存於 IndexedDB，不需後端、不上傳雲端
- 📱 **PWA 支援**：可加入 iOS / Android 主畫面，支援離線使用

## 指引與健保文件下載

### 健保給付規定下載在哪裡

App 內新增「來源中心」，可從首頁或底部導航進入：

- **健保第 9 節：抗癌瘤藥物**：腫瘤用藥給付最常用 PDF。
- **健保藥品給付規定：整份帶走**：需要完整章節時使用。
- **最新版藥品給付規定內容（分章節）**：官方更新後用來確認第 9 節最新版。

下載 PDF 後，回到「PDF 文件庫」→「+ 單筆」或「批次匯入」，來源填「健保給付規定」。

健保結果採保守呈現：有明確資料時顯示「健保有相關給付，但有限制條件」並逐項列出；條件欄位未填或規定尚未核對時顯示「資料不足／尚待核對」，不會直接判定為沒有給付。快速參考會合併手動資料、健保 PDF 自動擷取條目與 NCCN 療程成分，點選後只展開該項限制。

### NCCN 指引

NCCN 指引通常需要在 NCCN 官方網站登入後逐份下載。這個 app 目前不保存 NCCN 帳密、不自動登入、不批量抓取 NCCN PDF，避免違反登入與授權邊界。

可以自動化的部分是：

- 集中開啟 NCCN 官方 guideline 入口
- 提醒本機 NCCN PDF 超過 6 個月未確認版本
- 協助匯入、設為目前版本、封存舊版
- 在本機逐頁建立版本、章節代碼、治療相關頁及原始 PDF 頁碼索引
- 依目前輸入的病程、線別、病理與生物標記，排序優先查閱頁面並列出 PDF 擷取候選選項
- 發現互相矛盾的條件時先警示，不把候選選項顯示成最終治療決策

既有 PDF 不必重新匯入：到「PDF 文件庫」按「解析全部 NCCN」即可逐份補建索引。之後匯入新版時，照常設為目前版本；App 會把該份列為待解析，可按單份「解析 NCCN 結構」或再次按「解析全部 NCCN」。因此一般版次更新不需要再請開發者手動對齊；只有 PDF 版面改變而解析失敗，或需要新增更細的臨床條件判斷時才需調整程式。

## 操作說明

### 1. 初次使用

1. 開啟 App，會看到全癌別分類列表
2. 所有癌別預設為「待維護」狀態
3. 可直接搜尋癌別、開啟來源中心，或開始匯入指引 PDF

### 2. 匯入 PDF

1. 點擊「PDF 文件庫」→「+ 單筆」或「批次匯入」
2. 如不知道文件來源，先進「來源中心」開官方下載入口
3. 選擇已下載的指引 PDF
4. 填寫指引名稱、來源、版本
5. 選擇對應癌別（可多選）
6. 設為目前版本或歸檔

批次匯入在手機上每頁只顯示 10 份，並逐檔寫入。畫面會顯示實際收到的檔案數、總容量與瀏覽器估計可用空間；單一檔案失敗不會中止整批，重試時會略過已成功項目。若 OneDrive 回傳 0 位元或無法讀取的雲端檔案，請先下載到裝置後再重試失敗項目。

NCCN 批次匯入完成後，到文件庫按「解析全部 NCCN」。解析會逐份、逐頁在本機進行，單一文件失敗不會中止其餘文件；請保持 App 開啟。完成後每份文件卡會顯示版本、章節頁數與治療相關頁數。

解析器會把候選頁分為「具體建議、治療路徑、治療原則、檢查評估、補充資料」，快速參考優先顯示具體建議與治療路徑。拆出的選項同時標示手術、放射治療、藥物／全身治療或追蹤，並保留 NCCN 章節碼、PDF 頁碼及可解析的後續步驟連結。

「我要更精準」欄位有值時，App 會使用病程、治療線別、病理分類與分子標記比對候選；陰性標記會阻擋明顯衝突的療程。這仍是本機自動擷取的快速參考，需以原始 NCCN 流程圖、腳註及完整病況做最後核對。

真實 PDF 回歸檢查分成兩層：`audit:nccn` 精準驗證 HCC、NSCLC、SCLC、乳癌、直腸癌、前列腺癌及神經內分泌／腎上腺腫瘤；`audit:nccn-all` 會先比對 69 套預期 NCCN 指引並直接列出缺少名稱，清單齊全後才逐份檢查章節、治療頁、療程選項，以及常見或疾病專屬條件能否帶出全身治療。

整批 PDF 更新後可執行：

```powershell
$env:NCCN_PDF_DIR='C:\path\to\NCCN App PDFs'
npm run audit:nccn-all
```

### 3. 編輯癌別

1. 進入癌別詳情頁
2. 點擊「編輯此癌別」
3. 可編輯：
   - 基本資料（中英文名稱、同義詞）
   - 指引路由（主要/次要指引）
   - 必查章節（每行一筆）
   - 外部連結

### 4. 使用「不要漏」Checklist

1. 第一次開啟新版時，App 會保留既有內容並補入共同與本癌別專屬項目
2. 可自行勾選、編輯或刪除；刪除後不會在每次啟動時自動加回
3. 需要還原時，點「加入本癌別範本」只補回缺少的同名項目

### 5. 使用「我要更精準」欄位

1. 每癌別會預載共同欄位，以及適用的病理、分期、PD-L1、基因或風險欄位
2. 可新增、修改或刪除單選、多選、文字與數字欄位
3. 所有欄位可留白，不阻擋快速參考；範本只用來整理資料，不直接產生治療結論

### 6. 快速參考

1. 進入癌別 → 點擊「📋 快速參考」
2. 顯示：
   - 應查指引
   - 本機 PDF 狀態
   - 必查章節
   - 不要漏 checklist
   - 藥物／療程選擇、該項健保限制與缺漏條件補填
   - 目前資訊不足提醒
   - 下一步建議

快速參考中的補填值只保留在目前分頁，不會寫回癌別固定欄位；按「新個案／清除條件」會清除本次條件與選藥。

### 7. 備份與還原

1. 進入「設定」，先選「匯出資料備份（不含 PDF）」保存癌別卡、解析索引、checklist、精準欄位與健保資料。
2. 再選「分批備份 PDF」；App 每批最多處理 5 份 PDF，請逐批點擊下載，避免 Android 記憶體不足。
3. 還原時先「匯入資料備份」，再用「匯入 PDF 批次」一次選取一或多個批次檔補回 PDF。
4. 舊版 v1／v2 JSON 備份仍可匯入。

> ⚠️ 建議將本 App「加入主畫面」安裝使用，定期匯出資料備份，並完成全部 PDF 批次。
> App 會自動要求持久儲存（persistent storage），降低瀏覽器在
> 儲存空間不足時清除本機資料（含 PDF）的風險。

## 資料結構

所有資料儲存於瀏覽器 IndexedDB：

- `cancerCards`：癌別路由卡
- `documents`：PDF 文件元資料，以及 NCCN 的 `nccnStructure` 版本／章節／治療頁索引
- `checklists`：防漏 checklist 項目
- `precisionFields`：精準化欄位定義
- `nhiNotes`：藥物／療程健保資料

PDF 檔案本體儲存於獨立的 `OncologyPdfStorage` IndexedDB。

## 全癌別模板

內建 90+ 癌別模板，並對應 NCCN Treatment by Cancer Type 69 項分類。所有癌別都有共同防漏／精準欄位，主要實體腫瘤與血液腫瘤另有癌別專屬的病理、分期與分子標記範本；涵蓋：消化系、胸腔、乳癌、婦癌、泌尿系、頭頸癌、皮膚與黑色素細胞癌、骨與軟組織腫瘤、中樞神經系統腫瘤、血液腫瘤、內分泌與神經內分泌腫瘤等。

## 技術規格

- **前端**：Vanilla JavaScript（無框架依賴）
- **儲存**：IndexedDB（Dexie 風格 API）
- **PWA**：Service Worker + Manifest + iOS mobile web app meta
- **樣式**：CSS 變數 + 手機優先響應式設計
- **備份**：JSON 格式

## 注意事項

- 本 App 不是院級醫療資訊系統
- 不串接病人資料、不代替官方登入或批量下載受授權限制的指引
- 不輸出「最終治療建議」
- 不上傳資料到雲端
- 僅供個人查閱參考使用
